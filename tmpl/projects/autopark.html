Проект: Автопарк

У фармацевтической компании возникла потребность автоматизировать сбор и обработку отчетности по использованию корпоративного автотранспорта. Исходные данные:

  • есть медицинские представители в различных городах России, за каждым из которых закреплен корпоративный автомобиль;
  • каждый месяц водитель сдает отчетность о пробеге и заправленном топливе;
  • в конце месяца водитель заполняет путевой лист, в котором расписывает маршруты своих поездок.

Изначально в компании процесс был организован следующим образом. Были утвержденные формы в excel, которые заполнялись пользователями корпоративных автомобилей. Затем заполненные формы в виде файлов пересылались в центральный офис, где транспортный отдел вручную обрабатывал полученные файлы. Бизнес процесс состоял из следующих шагов:

  1. Собрать со всех сотрудников информацию (что само по себе растягивалось почти на месяц);
  2. Сверить расход бенизина с выпиской по топливным картам и с информацией из бухгалтерии по заправкам за наличный расчет;
  3. Свести все давнные в итоговый excel файл и передать его в бухгалтерию.

Цели внедрения:

  • повысить скорость обработки данных и устранить ошибки, неизбежные при ручной обработке excel файлов;
  • высвободить время продавцов, чтобы они не тратили как миниум один день на заполнение отчетности, а использовали это время для визитов по врачам и аптекам;
  • сбор информации в единую базу данных, с последующим извлечением различной аналитической и статистической информации для руководства.

На базе DSCommon было разработана и успешно внедрена система Автопарк, в которой была реализована следующая функциональность:

  • личный кабинет для каждого водителя, через который сотрудник осуществляет следующие операции:
    • ежемесячно вносит данные о пробеге;
    • вносит информацию об авариях;
    • отчитывается об оплате штрафов;
    • пишет заявки на ремонт (которые напрямую пересылаются в лизинговую компанию);
    • заполняет заявки на покупку расходных материалов;
  • автоматическая рассылка напоминание по email о необходимости заполнить ежемесячный отчет;
  • импорт данных из ПетролПлюс (поставщика топливных карт);
  • импорт данных из 1С о заправках за наличный расчет;
  • автоматическая генерация путевых листов;
  • автоматическое формирование отчетов для бухгалтерии в формате для импорта в 1С;
  • интеграция в корпоративную система AD авторизации.

За счет того что клиентские интерфейсы были реализованы на web технологиях (по сути являлись обычными web сайтами) это позволило без проблем подключить всех сотрудников (даже в самых удаленных городах России) без дополнительных усилий со стороны технических служб. Responsive верстка дизайна личного кабинета позволила сотрудникам работать с информацией как с ноутбуков, так и с планшетов различных моделей.
